# 月份唱片馆 · 使用手册

这是你的个人回忆网站。你不需要懂任何代码，只需要会两件事：往文件夹里放文件，以及编辑一个文本文件。

---

## 一、首次部署（只做一次，大约 10 分钟）

1. 登录 GitHub，点右上角 ＋ 号，选 **New repository**
2. 仓库名随意（比如 `my-memory`），选 **Public**，点 **Create repository**
3. 在新仓库页面点 **uploading an existing file**，把这个压缩包解压后的**所有内容**（包括 `.github` 这个隐藏文件夹）拖进去，点绿色的 **Commit changes**
   * 如果电脑上看不到 `.github` 文件夹，说明系统隐藏了它。Mac 在访达里按 Cmd+Shift+句号 显示隐藏文件；Windows 在文件夹选项里勾选"显示隐藏的项目"
4. 进仓库的 **Settings → Pages**，在 Source 处选 **Deploy from a branch**，Branch 选 **main**，点 **Save**
5. 进仓库的 **Settings → Actions → General**，在 Workflow permissions 处选 **Read and write permissions**，点 **Save**
6. 等两三分钟，回到 Settings → Pages，页面上方会出现你的网址，形如 `https://你的用户名.github.io/my-memory/`

这个网址就是你的唱片馆，可以存到手机浏览器书签里。

---

## 二、每个月更新（每次大约 10 分钟）

网站的全部内容都在 `memories` 文件夹里，结构是：

```
memories/
  2026/
    01/   ← 一月
    02/   ← 二月
    ...
    12/   ← 十二月
```

月底的时候，进入 GitHub 上你的仓库，打开对应月份的文件夹，点 **Add file → Upload files**，做三件事：

### 1. 放照片和视频
直接把文件拖进去，数量不限。照片和视频会自动出现在网页上，**文件名会变成照片下方的小标注**，所以起名时可以顺手写一句话，比如 `和朋友在海边看日落.jpg`。

### 2. 放这个月的歌
把一个音频文件（mp3、m4a 等）拖进同一个文件夹，它就是这个月的黑胶唱片。

### 3. 写月度总结
打开文件夹里的 `summary.txt`（点开文件后点右上角铅笔图标编辑），格式是：

```
歌名: 夜空中最亮的星
歌手: 逃跑计划

正文从这里开始写，想写多长写多长，
换行也没问题。
```

最后点绿色的 **Commit changes** 保存。等一两分钟，刷新你的网站，新内容就上线了。

## 三、小技巧

* **指定背景照片**：把某张照片改名为 `cover.jpg`（或 cover.png），它就会成为这个月的整页背景和唱片圆标。不设置的话，默认用文件夹里第一张照片。
* **开启新的一年**：在 `memories` 文件夹里新建一个年份文件夹（比如 `2027`），再在里面建 `01` 到 `12` 的月份文件夹即可。最省事的办法是把某个空月份的 summary.txt 复制过去。
* **视频尽量小**：GitHub 单个文件上限 100MB，建议视频控制在手机随手拍的长度，传之前用手机相册自带的压缩或"截取片段"功能处理一下。照片没有这个烦恼。
* **删除内容**：在 GitHub 上打开那个文件，点右上角的垃圾桶图标即可。
* **手机也能更新**：GitHub 网页版在手机浏览器上也能上传文件，出门在外也可以随手存一张。

## 四、出问题了怎么办

* **上传完网站没变化**：等两分钟再刷新。GitHub 每次上传后需要一点时间自动整理内容。
* **网页显示"还没读到内容清单"**：多半是第一步里的第 5 条（Actions 权限）没设置，去设置一下，然后随便上传一个文件触发更新。
* **照片不显示**：确认格式是 jpg、png、gif 或 webp。苹果手机的 HEIC 格式浏览器不认识，导出时选"最兼容"或先转成 jpg。
